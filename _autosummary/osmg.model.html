class Model(name: str, uid_generator: ~osmg.gen.uid_gen.UIDGenerator = <factory>, settings: ~osmg.model.Settings = <factory>)[source]¶

Model object. A model object is a representation of a structural model in OpenSees. It is the primary object of osmg. It contains levels, elastic sections, fiber sections, physical materials, and various collections of objects such as nodes, elements, and component assemblies that exist inside each level. Those objects are populated by generator objects. See osmg.gen.

component_connectivity() → dict[tuple[int, ...], ComponentAssembly][source]¶

Returns the connectivity of all component assemblies. Component assemblies are collections of lower-level components that are connected to primary nodes. Each component assembly can be represented by a tuple of node uids of its connected nodes in ascending order. This method returns a dictionary having these tuples as keys, and the associated components as values.

class Settings(imperial_units: bool = True, ndm: int = 3, ndf: int = 6)[source]¶
General customization of a model.
imperial_units:
True for imperial:

in, lb, lb/(in/s2)

False for SI:

m, N, kg

ndm, ndf: Number of dimensions and degrees of freedom.

Currently only ndm=3, ndf=6 is supported. This might be extended in the future if needed.
